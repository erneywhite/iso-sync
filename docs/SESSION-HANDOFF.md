# Session handoff — iso-sync

Сводка работы над проектом, чтобы любую сессию можно было продолжить без потери
контекста. Краткий «постоянный» контекст — в [../CLAUDE.md](../CLAUDE.md);
здесь — история, решения и обоснования.

> Документ публичный (репозиторий открытый). Операционные секреты
> (IP сервера, точные пути на проде) сюда намеренно НЕ входят — они живут
> в локальной авто-памяти Claude Code, не в гите.

---

## Хронология работы

1. **Рефакторинг ядра.** Логика вынесена в `lib/` (`namespace IsoSync`, PHP 8.1+,
   без Composer, ручной `bootstrap.php`). Конфиг переведён на JSON
   (`config/iso-list.json` + схема). Введены **4 режима записей**:
   - `fixed` — точное имя файла на сервере;
   - `latest` — фиксированное локальное имя, авто-выбор свежего внутри папки;
   - `family` — захват семейства версий, динамическое имя по шаблону + cleanup старых;
   - `discovery` — перебор папок по диапазону (Ubuntu LTS подхватывает будущие релизы).
   Добавлен самописный test-runner (`tests/run.php`) и README.

2. **Надёжность загрузки.** Второй backend `Aria2Downloader` (aria2c, multi-stream
   через HTTP Range, x2–x4 быстрее) за общим `DownloaderInterface`; cURL-resume
   через Range; ротация логов; чистка осиротевших `.tmp` на старте; `flock`.
   Атомарная подмена `*.tmp` → `rename()`. `mtime` = upstream `Last-Modified`
   (через `touch`), чтобы дата означала «релиз», а не «когда скачали».

3. **Сетевой кейс Proxmox.** Официальный CDN по IPv4 блокировался из хостинг-сети
   (TCP-таймаут при корректном DNS). Решение после диагностики: ходить на узел
   `de.cdn.proxmox.com` напрямую по **IPv6** (он в SAN серта, валиден), добавлен
   per-entry параметр **`ip_version: v4|v6|any`**. Для `v6`-записей загрузка идёт
   через cURL (чистый IPv6-only), не через aria2c. Зеркала-посредники отвергнуты:
   протухают относительно официального CDN.

4. **GPG.** Верификация подписи `SHA256SUMS` (`.sign`/`.gpg`/`.asc`) для
   Debian/Ubuntu. Требует импорта ключей в keyring под cron-пользователем.

5. **Модернизация веб-интерфейса.** Последовательными итерациями:
   неоново-фиолетовая палитра, mesh-gradient фон, glassmorphism, stagger-вход
   строк, hover-glow, inline-копирование хэша, **latest-бейджи по семействам**
   (а не один на папку), freshness-точка у даты, бренд-цвета дистрибутивов,
   bento-статус из 3 карточек, counter-up на тоталах, «История обновлений»
   из логов, поиск по имени **и SHA256-хэшу** с крестиком очистки,
   иллюстрированный empty-state, **sparkline** роста хранилища с дельтами 7д/30д
   и draw-on прорисовкой. Все анимации — за `prefers-reduced-motion`.

6. **Фикс «моргания» на Firefox/Mac.** Trail-артефакты под карточками. Перебрали
   гипотезы (`backdrop-filter`, `mix-blend-mode`, `contain`/`isolation`). Ключевая
   подсказка пользователя — артефакты уходили через ~20 c (≈ треть цикла анимации
   фона). Корень — движущийся `transform`-слой нейбулы под UI. Итог: анимация
   переведена на `filter: hue-rotate` (геометрия не меняется → композитор стабилен,
   «живость» сохранена). По пути убраны `backdrop-filter` с bento и `mix-blend-mode`
   с грейна как лишние раздражители на проблемных браузерах.

7. **Оформление репозитория.** Полная переработка README (hero, GitHub-callouts,
   свёрнутые `<details>`, таблица дистрибутивов), приведён в порядок `crontab`,
   заполнен About на GitHub (description, homepage, topics), добавлен CI-workflow
   (`Tests` на PHP 8.1–8.4) и зелёный бейдж.

8. **Переезд каталога** (2026-05-31). Проект перенесён с рабочего стола в
   `Documents\ClaudeCode\` (имя папки сохранено), добавлены этот handoff и
   `CLAUDE.md`.

9. **Фикс перекрытия тултипов/меню** (2026-06-10). Тултипы «Скачать файл» и
   dd-menu «Копировать» резались по границе ряда и уходили под соседние ряды
   (клик в зону нависающего меню попадал в соседний ряд). Корень —
   `contain:paint` на `.row` (клип потомков + свой stacking context у каждого
   ряда). Фикс: на время взаимодействия (hover / focus-within / menu-open)
   containment снимается, ряд поднимается z-index'ом (10; с открытым меню 11,
   чтобы ховер соседа не перекрывал пункты меню). Проверено браузерным репро
   «до/после» на реальном CSS и адверсариальным ревью: Mac-ghost не
   ре-армируется (фон статичен по геометрии). Попутно вычищены вскрывшиеся
   клавиатурные баги: Enter на кнопках меню запускал скачивание (бабблинг
   keydown до ряда), мёртвый клик по стрелке-toggle папки, протухающий
   `aria-expanded` при закрытии меню не через кнопку. Вторым заходом добиты
   кромки скролл-порта `.card`: тултип у верхней кромки флипается вниз
   (`.tip-down` по замеру в делегированном mouseover, порог 48px с учётом
   hover-lift ряда и магнитного смещения кнопки), открытое меню закрывается
   при прокрутке карточки (гард 150мс — от мгновенного захлопывания
   браузерным scroll-into-view при клике по полусрезанной кнопке), фокус из
   закрываемого меню возвращается на кнопку-тогглер.

## Ключевые решения и обоснования

- **JSON-конфиг + 4 режима** вместо плоского списка — покрывает реальные схемы
  публикации апстримов (от фиксированного имени до автоподхвата будущих LTS).
- **aria2c как опциональный ускоритель**, не зависимость — при отсутствии в PATH
  прозрачный фолбэк на тюнингованный cURL. Целостность (SHA256 + Content-Length)
  проверяется ПОСЛЕ загрузки в любом случае.
- **Proxmox через IPv6-узел CDN** — единственный способ получить свежий
  официальный контент из заблокированной по IPv4 сети, без перехода на
  протухающие зеркала.
- **`filter: hue-rotate` для фона** вместо `translate` — единственный вариант,
  дающий «движение» без trail-артефактов на Firefox/Mac.

## Конвенции работы над проектом

- **Источник данных / инфраструктура** (зеркала, URL, способ доставки) меняются
  только после показа вариантов с trade-offs и явного апрува — не сразу коммитом.
- Правки кода / UI / багфиксы — обычным порядком, можно сразу.
- Внешние источники проверять на **свежесть И достижимость с конкретного сервера**.
- **Privacy**: server-paths не выводятся в UI — «История» показывает только
  basename, не локальный путь.
- Диагностика прода — самодостаточным скриптом (его запускают и присылают вывод),
  без прямого доступа к боевой машине.

## Отложенные идеи

- Drag-drop проверка хэша через Web Crypto API (сверка SHA256 в браузере).
- Сплит `index.php` на отдельные `assets/style.css` + `assets/app.js` (`?v=<hash>`).
- Флаг `--only=<entry>` у `update_iso.php` для точечного обновления.
